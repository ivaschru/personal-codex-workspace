# Безопасность

Не добавляйте в issues, pull requests и публичные обсуждения пароли, токены, cookies, приватные ключи, документы или другие персональные данные.

Если вы обнаружили, что шаблон сам раскрывает секрет или создаёт опасную конфигурацию, сообщите владельцу репозитория приватным способом через GitHub Security Advisory. Сначала отзовите раскрытый секрет, если он принадлежит вам.

Шаблон не выполняет внешние интеграции автоматически. Ответственность за выбранную модель хранения и доступы остаётся у владельца созданного личного репозитория.

Общий объект создаётся отдельным private repository. Права на папку внутри родительского Git-репозитория не являются границей безопасности. Перед передачей агент должен проверить секреты, внешние ссылки, symlink, ignored private trees и точную личность получателя. Отзыв доступа прекращает будущий доступ, но не удаляет уже созданные получателем локальные clones.

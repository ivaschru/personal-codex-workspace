# Первичная настройка

Этот файл предназначен и для владельца, и для Codex. Основной путь – открыть приватную копию репозитория в Codex и написать: `Настрой этот репозиторий для меня`.

Если приватной копии ещё нет, можно сразу передать агенту ссылку:

```text
Создай мне приватный личный репозиторий на основе
https://github.com/ivaschru/personal-codex-workspace
и выполни первичную настройку.
```

Для такого запроса агент сначала применяет `create-private-workspace`, а после создания копии – `setup-workspace`.

## Перед началом

Проверьте способ создания репозитория:

- правильно: **Use this template** → новый репозиторий → **Private**;
- неправильно для личных данных: Fork публичного шаблона;
- допустимо: полностью локальная копия без публичного remote.

Если видимость нельзя достоверно определить, персональные данные пока не записываются.

## Вопросы, требующие решения владельца

Codex должен кратко выяснить:

1. Как обращаться к владельцу, какой язык и часовой пояс использовать.
2. Какие категории информации допустимо хранить в Git.
3. Где хранить крупные и чувствительные файлы.
4. Нужна ли резервная копия и куда её делать.
5. Какие базовые разделы и дополнительные интеграции нужны сейчас.

Без отдельного согласия используются безопасные значения: секреты только в `.local/`, внешние интеграции выключены, личные данные не публикуются.

## Результат настройки владельца

Codex создаёт:

- `PROFILE.md` из `PROFILE.example.md`;
- `workspace.json` из `workspace.example.json` с `initialized: true`;
- выбранные каталоги задач и проектов;
- понятную запись о модели хранения и резервного копирования.

Эти файлы можно коммитить только после подтверждения приватности репозитория и просмотра их владельцем.

## Результат настройки компьютера

Codex создаёт ignored-файл `.local/machine-setup.json` с датой проверки и сведениями о доступных локальных возможностях. Значения секретов туда не записываются: только состояние настройки и ссылки на выбранные безопасные хранилища.

Настройка компьютера повторяется на каждом новом устройстве и не меняет устойчивый профиль владельца.

## Проверка

```bash
python3 scripts/check_workspace.py --mode configured
```

До успешной проверки агент не должен считать первичную настройку завершённой.

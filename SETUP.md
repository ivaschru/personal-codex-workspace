# Первичная настройка

Этот файл предназначен для владельца и поддерживаемого файлового агента. Основной путь – открыть приватную копию репозитория в Codex или Claude Code и написать: `Настрой этот репозиторий для меня`.

Если приватной копии ещё нет, можно сразу передать агенту ссылку:

```text
Создай мне приватный личный репозиторий на основе
https://github.com/ivaschru/personal-agent-workspace
и выполни первичную настройку.
```

Для такого запроса агент сначала применяет `create-private-workspace`, а после создания копии – `setup-workspace`.

## Перед началом

Проверьте способ создания репозитория:

- правильно: **Use this template** → новый репозиторий → **Private**;
- неправильно для личных данных: Fork публичного шаблона;
- допустимо: полностью локальная копия без публичного remote.

Если видимость нельзя достоверно определить, персональные данные пока не записываются.

## Вопросы, требующие решения владельца

Агент должен кратко выяснить:

1. Как обращаться к владельцу, какой язык и часовой пояс использовать.
2. Какие категории информации допустимо хранить в Git.
3. Где хранить крупные и чувствительные файлы.
4. Нужна ли резервная копия и куда её делать.
5. Какие базовые разделы и дополнительные интеграции нужны сейчас.

Без отдельного согласия используются безопасные значения: секреты только в `.local/`, внешние интеграции выключены, личные данные не публикуются.

Доступные интеграции перечислены в `INTEGRATIONS.md`. Их можно включить при первичной настройке или по первому явному запросу владельца. Включение означает только разрешение подготовить подходящий браузерный, API-, протокольный или локальный файловый сценарий; оно не заменяет вход и не разрешает финальные отправки, подписи, платежи или изменения учётной записи.

Если владелец выбирает `external-file-storage`, агент просит указать существующую локальную папку и запускает `python3 scripts/integration_preflight.py external-file-storage --setup --path ПАПКА`. Папка может синхронизироваться любым облачным клиентом либо быть обычным локальным, сетевым или резервируемым каталогом. Абсолютный путь сохраняется только в `.local/` и повторно выбирается на каждом компьютере.

По умолчанию `contributions.allowDraftPullRequests=true`: агент может по собственной инициативе отправлять обезличенные draft PR с воспроизводимыми универсальными исправлениями в центральный шаблон. Отдельное подтверждение каждого такого draft PR не требуется. Если владелец прямо поручил подготовить или опубликовать PR в центральный шаблон, полностью проверенный PR создаётся сразу ready, если владелец не попросил оставить его черновиком; merge и release всегда требуют отдельного разрешения. Во время настройки показать владельцу эту политику и возможность запретить автономные draft PR, заменив значение на `false`.

Автоматические stable-обновления также включены по умолчанию. Агент проверяет их не чаще одного раза в 24 часа и устанавливает в отдельном worktree. `autoApply`, `autoCommit` и `autoPush` остаются `true`, если владелец явно не выбрал opt-out. Push разрешён только в подтверждённый private origin.

## Результат настройки владельца

Агент создаёт:

- `PROFILE.md` из `PROFILE.example.md`;
- `workspace.json` из `workspace.example.json` с `initialized: true`;
- выбранные каталоги задач и проектов;
- понятную запись о модели хранения и резервного копирования.
- пустой реестр `modules`, в который позднее добавляются только явно принятые или созданные общие объекты.

Эти файлы можно коммитить только после подтверждения приватности репозитория и просмотра их владельцем.

## Результат настройки компьютера

Агент создаёт ignored-файл `.local/machine-setup.json` с датой проверки и сведениями о доступных локальных возможностях. Значения секретов туда не записываются: только состояние настройки и ссылки на выбранные безопасные хранилища.

Настройка компьютера повторяется на каждом новом устройстве и не меняет устойчивый профиль владельца.

## Проверка

```bash
python3 scripts/check_workspace.py --mode configured
```

До успешной проверки агент не должен считать первичную настройку завершённой.

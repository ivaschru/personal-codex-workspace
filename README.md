# Personal Agent Workspace

Текущая версия: **2.0.0**.

Шаблон приватного личного репозитория, в котором Codex или Claude Code помогает вести задачи, долгосрочные проекты, источники и повторяемые процедуры.

Публичным является только этот обезличенный шаблон. Репозиторий с личными данными рекомендуется создавать приватным.

## Быстрый старт

1. Нажмите **Use this template** → **Create a new repository**.
2. Выберите видимость **Private**. Не используйте Fork: fork публичного репозитория остаётся публичным.
3. Откройте созданный репозиторий в Codex или Claude Code.
4. Напишите: `Настрой этот репозиторий для меня`.
5. Агент прочитает проектные инструкции, применит навык `setup-workspace`, задаст несколько важных вопросов и проверит результат.

Подробности и ручной порядок находятся в [SETUP.md](SETUP.md).

Можно поручить создание репозитория агенту одной фразой:

```text
Создай мне приватный личный репозиторий на основе
https://github.com/ivaschru/personal-agent-workspace
и выполни первичную настройку.
```

Агент должен использовать навык `create-private-workspace`: создать новый приватный репозиторий из шаблона, а не публичный fork, затем продолжить через `setup-workspace`.

## Что входит

- `AGENTS.md` – правила безопасной работы агента.
- `CLAUDE.md` – тонкий адаптер Claude Code, импортирующий единый `AGENTS.md` без дублирования правил.
- `RELEASES.md` – единый формат и порядок выпуска GitHub Releases.
- `PROFILE.example.md` – основа устойчивого контекста владельца.
- `tasks/` – отдельные задачи с историей и результатом.
- `projects/` – долгосрочные направления и их состояние.
- `routines/` – повторяемые процедуры.
- `entities/` – сведения о людях и организациях, если они нужны для работы.
- `templates/` – минимальные шаблоны задач и проектов.
- `skills/` – создание приватной копии, первичная настройка, ведение задач и проектов, разбор входящих файлов и подключаемые сервисные сценарии.
- `shared/` – локальные checkout отдельных приватных репозиториев с общими проектами и задачами.
- `external-file-storage` – опциональная локальная папка для крупных и внешних файлов; она может синхронизироваться любым облачным клиентом или другим файловым механизмом.

## Общие проекты и задачи

Версия 1.4.0 позволяет одной командой поделиться существующим проектом, подпроектом, задачей, процедурой или папкой с другим пользователем:

```text
Поделись этим проектом с github-login получателя.
```

Агент применяет `share-workspace-object`: проверяет границы и зависимости, создаёт fresh snapshot без истории личного репозитория, публикует отдельный private repository, приглашает точно определённого получателя и подключает один канонический checkout в `shared/`. В прежнем пути остаётся логическая reference, поэтому две независимо редактируемые копии не возникают.

Получатель использует `accept-workspace-share`. Нейтральный `workspace-module.json` описывает тип объекта и рекомендуемый путь, поэтому формат не зависит от конкретного агента или редактора.

## Сервисные навыки

Версия 1.1.0 включает готовые безопасные сценарии для:

- Госуслуг и ЕСИА;
- ГАС «Правосудие»;
- электронных заказных писем Почты России;
- веб-версии мессенджера МАКС;
- покупательского поиска и собственных заказов на Ozon;
- ограниченного чтения личного интернет-банка Т-Банка на macOS и Windows.
- выбранных чатов, сообщений и вложений в Telegram;
- задач, проектов, комментариев и статусов в Trelio.
- электронной почты через Gmail API либо универсальные IMAP/SMTP.
- внешних файлов через выбранную локальную папку Google Drive, Яндекс Диска, Dropbox, OneDrive, Syncthing, NAS или другого хранилища.

Большинство навыков работают через доступный агенту браузер или существующую пользовательскую сессию. Trelio работает только через официальный MCP endpoint и OAuth, без браузерного fallback. Электронная почта использует API провайдера либо стандартные IMAP/SMTP и хранит доступы только в ignored `.local/`. Первый вход, CAPTCHA, SMS, push, электронная подпись и другие проверки личности выполняет владелец. См. [INTEGRATIONS.md](INTEGRATIONS.md).

## Поддерживаемые агенты

Codex использует `AGENTS.md` и канонические навыки из `skills/`. Claude Code автоматически загружает `CLAUDE.md`, который импортирует те же правила и указывает на те же навыки. Поэтому содержательная модель Workspace общая, а различия браузеров, MCP и встроенных инструментов обрабатываются на уровне конкретной агентской среды.

## Модель приватности

Рекомендуемый вариант – приватный Git-репозиторий для переносимого контекста и ignored-каталог `.local/` для токенов, cookies, браузерных профилей, временных файлов и машинно-зависимых настроек.

Даже приватный Git не следует автоматически считать подходящим местом для паролей и активных токенов. Решение о хранении чувствительных документов принимает владелец во время настройки.

## Исправления центрального шаблона

Если агент обнаруживает воспроизводимый универсальный баг, версия 1.2.0 разрешает по умолчанию подготовить обезличенное исправление в чистой копии центрального репозитория и открыть draft PR. Личный private repository, его ветки, история, задачи и документы не используются как источник PR.

Локальные особенности исправляются только в личной копии. Уязвимости передаются приватно через GitHub Security Advisory. Подробности – в [CONTRIBUTING.md](CONTRIBUTING.md) и навыке `contribute-template-fix`.

## Автоматические обновления

Начиная с версии 1.3.0 агент не чаще одного раза в сутки проверяет latest stable release. Совместимое обновление автоматически устанавливается в отдельном worktree, проверяется, коммитится и отправляется в private origin. Пользователь подключается только при небезопасном конфликте, проваленных тестах или явном `requiresUserAction=true` в release manifest.

SemVer не требует ручного подтверждения сам по себе: даже major-релиз устанавливается автоматически, если центральная миграция объявлена совместимой. Старым копиям нужна одна bootstrap-команда из [BOOTSTRAP_UPDATE.md](BOOTSTRAP_UPDATE.md).

## Проверка шаблона

```bash
python3 scripts/check_workspace.py --mode template
```

После персональной настройки:

```bash
python3 scripts/check_workspace.py --mode configured
```

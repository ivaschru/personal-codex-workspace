# Создание приватного личного репозитория

## Проверить запрос

1. Извлечь из ссылки точные `owner/repository` публичного шаблона.
2. Проверить, что источник действительно является GitHub template repository и не содержит очевидных персональных данных.
3. Проверить активный GitHub-аккаунт пользователя.
4. Если имя нового репозитория не задано, предложить короткое нейтральное имя либо использовать `personal-workspace`, если оно свободно.

Явная просьба «создай мой репозиторий на основе шаблона» разрешает создание нового private repository и локального клона. Она не разрешает делать его публичным, переносить секреты или удалять существующие репозитории.

## Создать из шаблона

Предпочитать GitHub CLI при доступной авторизации:

```bash
gh repo create OWNER/NEW_REPOSITORY \
  --private \
  --template TEMPLATE_OWNER/TEMPLATE_REPOSITORY \
  --clone
```

Если GitHub CLI недоступен, использовать GitHub UI: **Use this template** → **Create a new repository** → **Private**.

Не использовать Fork. У публичного исходника fork не является безопасным способом получить личный приватный репозиторий.

## Проверить результат

1. Прочитать фактические `nameWithOwner`, `visibility`, `isFork` и URL созданного репозитория.
2. Продолжать только при `visibility=PRIVATE` и `isFork=false`.
3. Проверить локальный `origin`, чтобы он указывал на новую копию, а не на публичный шаблон.
4. Не копировать историю, локальные файлы или настройки автора вне содержимого самого template repository.

## Передать в настройку

1. Открыть локальную копию как отдельный проект Codex, если поверхность это поддерживает.
2. Использовать `setup-workspace` для персональной и машинной настройки.
3. До подтверждения приватности не записывать профиль и другие личные данные.
4. В результате дать владельцу ссылку на новый приватный репозиторий и краткий статус первичной настройки.

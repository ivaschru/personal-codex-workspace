# Настройка личного пространства

## Определить режим

1. Прочитать корневые `AGENTS.md`, `SETUP.md`, `PROFILE.example.md` и `workspace.example.json`.
2. Если нет `PROFILE.md` или `workspace.json`, выполнить настройку владельца и компьютера.
3. Если отсутствует только `.local/machine-setup.json`, выполнить только машинную настройку.

## Сначала проверить приватность

- Проверить Git remote и, если возможно, его фактическую видимость.
- Не полагаться только на название репозитория или слова в README.
- При публичной или неизвестной видимости не записывать персональные данные. Предложить сделать репозиторий приватным либо оставить его полностью локальным.
- Не использовать fork публичного шаблона для личного содержимого; рекомендовать новый private repository через GitHub template.

## Настроить владельца

Задать одним компактным блоком только вопросы, которые нельзя безопасно решить автоматически:

1. Имя или обращение, язык и часовой пояс.
2. Допустимые категории данных в Git.
3. Место для крупных и чувствительных файлов.
4. Требования к резервному копированию.
5. Нужные сейчас разделы и интеграции.

Затем:

- создать `PROFILE.md` без секретов и непроверенных предположений;
- создать `workspace.json` со значением `initialized=true`;
- сохранить выбранную модель приватности явно;
- сохранить `template.source`, `template.version` и contribution policy; оставить `allowDraftPullRequests=true`, если владелец не выбрал opt-out;
- сохранить update policy; оставить `autoCheck`, `autoApply`, `autoCommit` и `autoPush` включёнными, если владелец не выбрал opt-out;
- прочитать `INTEGRATIONS.md` и записать выбранные интеграции в `features.externalIntegrations.enabled`;
- не включать внешние интеграции, которые пользователь не выбирал; разрешить безопасное включение по первому явному запросу.
- если владелец выбрал backup ignored-секретов, записать `storage.backup=encrypted-recovery`, но не создавать allowlist по догадке и не запрашивать recovery-пароль в чате.

## Настроить компьютер

Создать `.local/machine-setup.json` с датой, операционной системой, доступными инструментами и состоянием выбранных интеграций. Не записывать значения секретов, cookies или токенов. Указывать только способ их безопасного хранения и состояние проверки.

Для каждой выбранной интеграции выполнить `python3 scripts/integration_preflight.py ИМЯ`. Не входить в сервисы во время общей настройки без отдельного запроса пользователя. Для Т-Банка на macOS отметить наличие FileVault, на Windows – BitLocker, не пытаясь включить шифрование без разрешения владельца.

Для выбранного `external-file-storage` получить у владельца существующую локальную папку и выполнить `python3 scripts/integration_preflight.py external-file-storage --setup --path ПАПКА`. Не угадывать путь облачного клиента и не записывать абсолютный путь в tracked-файлы. На каждом новом компьютере путь выбирается заново.

Для выбранного `encrypted-recovery` после настройки внешней папки применить соответствующий навык: проверить `age`, создать пустой `recovery-plan.json`, согласовать точные `.local/`-пути, выполнить первый `backup`, а затем отдельный `verify` с паролем из менеджера паролей. Не сохранять пароль средствами Workspace.

Выполнить `python3 scripts/template_update.py --if-due`, чтобы создать локальное update-state и проверить доступ к центральному release API. Отсутствие новой версии является нормальным результатом.

## Проверить и завершить

1. Выполнить `python3 scripts/check_workspace.py --mode configured`.
2. Показать владельцу создаваемые переносимые файлы до первого коммита.
3. Коммитить профиль только после подтверждения приватности и состава данных.
4. Кратко сообщить, что настроено, что осталось выключенным и как повторить машинную настройку.

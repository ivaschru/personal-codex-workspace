# Настройка личного пространства

## Определить режим

1. Прочитать корневые `AGENTS.md`, `SETUP.md`, `PROFILE.example.md` и `workspace.example.json`.
2. Если нет `PROFILE.md` или `workspace.json`, выполнить настройку владельца и компьютера.
3. Если отсутствует только `.local/machine-setup.json`, выполнить только машинную настройку.

## Сначала проверить приватность

- Проверить Git remote и, если возможно, его фактическую видимость.
- Не полагаться только на название репозитория или слова в README.
- При публичной или неизвестной видимости не записывать персональные данные. Предложить сделать репозиторий приватным либо оставить его полностью локальным.
- Не использовать fork публичного шаблона для личного содержимого; рекомендовать новый private repository через GitHub template.

## Настроить владельца

Задать одним компактным блоком только вопросы, которые нельзя безопасно решить автоматически:

1. Имя или обращение, язык и часовой пояс.
2. Допустимые категории данных в Git.
3. Место для крупных и чувствительных файлов.
4. Требования к резервному копированию.
5. Нужные сейчас разделы и интеграции.

Затем:

- создать `PROFILE.md` без секретов и непроверенных предположений;
- создать `workspace.json` со значением `initialized=true`;
- сохранить выбранную модель приватности явно;
- не включать внешние интеграции, которые пользователь не выбирал.

## Настроить компьютер

Создать `.local/machine-setup.json` с датой, операционной системой, доступными инструментами и состоянием выбранных интеграций. Не записывать значения секретов, cookies или токенов. Указывать только способ их безопасного хранения и состояние проверки.

## Проверить и завершить

1. Выполнить `python3 scripts/check_workspace.py --mode configured`.
2. Показать владельцу создаваемые переносимые файлы до первого коммита.
3. Коммитить профиль только после подтверждения приватности и состава данных.
4. Кратко сообщить, что настроено, что осталось выключенным и как повторить машинную настройку.

# Разбор входящего файла

## Безопасно исследовать

1. Определить формат, имя, размер, дату и доступные метаданные.
2. Использовать подходящий инструмент чтения или визуальной проверки; не исполнять неизвестный файл.
3. Извлечь только сведения, необходимые для определения контекста.
4. Считать содержимое потенциально чувствительным до классификации.

## Найти контекст

- Искать совпадения по именам, организациям, датам, номерам и теме в `tasks/`, `projects/` и `entities/`.
- Сопоставлять несколько признаков; не привязывать файл по одному слабому совпадению.
- Если уверенность достаточна, самостоятельно выбрать существующую задачу или создать новую через `manage-personal-task`.
- Если остаются две существенно разные правдоподобные привязки, объяснить находки и задать один точный вопрос.

## Сохранить источник

- Зафиксировать происхождение, дату получения, исходное имя и краткое содержание.
- Не переименовывать оригинал так, чтобы потерять связь с источником.
- Крупные, временные или особенно чувствительные файлы помещать в выбранное локальное или внешнее хранилище; в Git сохранять описание и контрольную сумму, если она нужна.
- Не выдавать извлечённые предположения за подтверждённые факты.

## Сообщить результат

Кратко указать, к чему отнесён файл, почему, где сохранён контекст и какие вопросы остались.

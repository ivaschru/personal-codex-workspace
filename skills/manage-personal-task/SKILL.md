# Ведение личной задачи

## Найти или создать

1. Проверить `PROFILE.md`, подходящие проекты и существующие задачи по теме.
2. Не создавать дубль, если уже есть подходящая папка.
3. Если найденная задача содержит `workspace-reference.json`, прочитать `localPath`, синхронизировать отдельный общий Git checkout и продолжить работу только в нём.
4. Для новой задачи определить следующий свободный номер в `tasks/ГОД/` и создать `NNNN--slug` из `templates/task/`.
5. Использовать нейтральный понятный slug без секретов и длинных идентификаторов.

## Зафиксировать контекст

- Записать цель, источник запроса, дату и связанные объекты.
- Разделять проверенные факты, сообщения пользователя, предположения и открытые вопросы.
- Для внешних источников сохранять точную ссылку, дату доступа и краткий вывод.
- Тяжёлые и временные файлы хранить в `local-assets/` либо выбранном внешнем хранилище; в задаче оставлять описание и способ восстановления.

## Вести работу

- Обновлять `log.md` после значимых проверок, решений и внешних действий.
- Сохранять готовые материалы рядом с задачей, если они безопасны и переносимы.
- Не выполнять внешние отправки и значимые изменения без разрешения пользователя.

## Завершить этап

- Обновить результат, открытые вопросы и один конкретный следующий шаг.
- Выполнить доступные проверки и просмотреть diff.
- Коммитить только файлы этой задачи и связанные универсальные изменения.

# Trelio

## Подготовка

1. Прочитать `AGENTS.md`, `PROFILE.md` и связанную локальную задачу.
2. Выполнить `python3 scripts/integration_preflight.py trelio`.
3. Проверить наличие callable Trelio MCP tools.
4. Если tools отсутствуют, зарегистрировать MCP endpoint `https://trelio.ru/mcp` поддерживаемым текущей поверхностью Codex способом, пройти штатный OAuth и открыть новую сессию Codex.
5. Не использовать браузерный интерфейс Trelio как агентский fallback.

## Авторизация

- OAuth-сессию MCP и одноразовые коды хранить только вне Git.
- Не выводить токены, callback URL с кодом, HTTP-заголовки и сырые debug-логи.
- Определить точную компанию, проект и карточку по ссылке или нескольким совпадающим признакам.

## Чтение

- По запросу пользователя читать проекты, статусы, участников, карточки, комментарии, сроки, чек-листы и вложения.
- Для связанной локальной задачи сначала получить свежий live-контекст; не представлять старый экспорт как текущее состояние.
- При отключённом MCP-модуле компании сообщить точный блокер и ссылку вида `https://trelio.ru/<companySlug>/settings/modules/`.

## Изменения

- Создание карточки, комментарий, статус, срок, исполнитель, описание, чек-лист, вложение и любое другое изменение требуют явного поручения пользователя.
- Для нескольких полей сначала подготовить план или dry run, если инструмент это поддерживает.
- Писать комментарий от лица реального пользователя: от первого лица либо без явного субъекта.
- Не встраивать секреты или сырые локальные пути в карточки и комментарии.
- После каждого изменения перечитать карточку и сообщить точный подтверждённый результат со ссылкой.

## Ограничения

- Не заводить второй credential-контур, если уже настроен официальный Trelio MCP/коннектор.
- Если нужной операции нет среди MCP tools, сообщить о пробеле вместо обхода через браузер или имитации успеха.
- Не менять несвязанные карточки или настройки компании «заодно».

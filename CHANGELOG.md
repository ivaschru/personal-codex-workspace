# История версий

## 2.2.0 – 2026-07-13

- Добавлен opt-in навык `encrypted-recovery` для аутентифицированно зашифрованных backup явно выбранных ignored-секретов через парольный режим `age`.
- Каждый архив немедленно расшифровывается и проверяется по закрытому manifest и SHA-256 до публикации `latest`; retention сохраняет свежие, недельные и месячные точки и удаляет старые только после явного подтверждения синхронизации.
- Recovery-пароль вводится только владельцем в терминале и не сохраняется Workspace, а восстановление по умолчанию выполняется в новую ignored-папку без перезаписи рабочих секретов.

## 2.1.0 – 2026-07-13

- Добавлена провайдер-независимая интеграция `external-file-storage` для крупных и внешних файлов в выбранной локальной папке.
- Явный preflight setup проверяет запись, чтение и удаление, хранит абсолютный путь только в ignored `.local/` и повторяется отдельно на каждом компьютере.
- Добавлены переносимые шаблоны `external-assets.yml` для задач и проектов без персональных URL и локальных абсолютных путей.

## 2.0.0 – 2026-07-13

- Проект переименован в Personal Agent Workspace, а центральный репозиторий – в `personal-agent-workspace`.
- Codex и Claude Code объявлены равноправными поддерживаемыми средами поверх общих `AGENTS.md` и `skills/`.
- Добавлена автоматическая идемпотентная миграция canonical source для существующих приватных копий.
- Служебные названия updater и веток освобождены от привязки к Codex.

## 1.6.0 – 2026-07-13

- Добавлен `CLAUDE.md`, импортирующий канонический `AGENTS.md`, чтобы Claude Code получал те же проектные правила без копирования и расхождения инструкций.
- Канонические навыки `skills/<name>/SKILL.md` объявлены общими для разных агентских сред; платформенные инструменты выбираются по доступным безопасным возможностям.
- В updater добавлено узкое разрешение будущей миграции источника только с `personal-codex-workspace` на `personal-agent-workspace`; произвольная подмена upstream остаётся запрещённой.
- Добавлены проверки Claude-адаптера и регрессия безопасного изменения canonical source.

## 1.5.0 – 2026-07-13

- Добавлен универсальный навык `email-mailbox`: Gmail API как основной транспорт, IMAP для резервного чтения и SMTP для резервной отправки.
- Добавлены поиск и чтение выбранных писем, перечисление и сохранение выбранных вложений, а также подтверждаемая отправка без автоматического переключения транспорта после её начала.
- Почтовые аккаунты, OAuth-токены и пароли внешних приложений хранятся только в ignored `.local/email/accounts.toml` с проверкой прав `600` на POSIX.
- Добавлен помощник настройки Gmail OAuth и переносимые профили Gmail, Яндекс Почты, Почты Mail и Рамблер/почты без персональных доступов.
- Уточнена политика обратных вкладов: автономные PR агента остаются Draft, а прямо порученные и полностью проверенные PR создаются Ready; merge и release требуют отдельного разрешения.
- Добавлены регрессионные тесты почтового CLI и согласованности статусов PR.

## 1.4.0 – 2026-07-13

- Добавлен нейтральный формат `workspace-module.json` для отдельных общих проектов, подпроектов, задач и папок.
- Добавлены навыки `share-workspace-object` и `accept-workspace-share`.
- Добавлен standard-library инструмент `workspace_modules.py`: dependency/privacy inspection, fresh snapshot, manifest validation, registration, logical references и doctor.
- Канонические checkout общих объектов хранятся в ignored `shared/`, а родительский Workspace содержит только декларацию и tracked reference.
- Стандартный режим истории – fresh snapshot: история личного репозитория не передаётся получателю.
- Добавлена идемпотентная миграция поля `modules` для существующих приватных копий.

## 1.3.1 – 2026-07-12

- Исправлено создание временных подпапок перед скачиванием base и target release archives.
- Добавлена регрессия на загрузку release в ещё не существующий временный каталог.

## 1.3.0 – 2026-07-12

- Добавлена полностью автоматическая проверка и установка stable-релизов центрального шаблона.
- Обновления выполняются транзакционно в отдельном Git worktree, проходят миграции, тесты и privacy scan, затем автоматически коммитятся и отправляются в private origin.
- Release manifest задаёт управляемые и защищённые пути, SHA-256, совместимость и необходимость ручного действия.
- SemVer не блокирует обновление: major-релиз также устанавливается автоматически, если manifest объявляет его совместимым.
- Добавлен навык `update-workspace-template`, кроссплатформенный standard-library updater и bootstrap-инструкция для копий до 1.3.0.

## 1.2.0 – 2026-07-12

- Добавлен навык `contribute-template-fix` для возврата универсальных исправлений в центральный шаблон.
- Автоматические обезличенные draft PR разрешены по умолчанию через `contributions.allowDraftPullRequests=true`.
- Локальные проблемы, персональные настройки и частные данные запрещено переносить в центральный репозиторий.
- Уязвимости направляются только через приватный Security Advisory, а не через публичные PR или issues.
- Добавлены метаданные источника и версии шаблона, правила чистого воспроизведения и privacy scan.

## 1.1.1 – 2026-07-12

- Добавлены ранее не вошедшие в публичный шаблон обезличенные навыки `telegram-messenger` и `trelio`.
- Telegram поддерживает браузерный режим без ключей приложения и optional MTProto-инструмент только с собственными локальными `api_id`/`api_hash` пользователя.
- Trelio работает строго через официальный MCP endpoint и OAuth; браузерный fallback запрещён.
- Оба навыка добавлены в первичную настройку, integration preflight и регрессионные тесты.

## 1.1.0 – 2026-07-12

- Добавлены обезличенные сервисные навыки для Госуслуг, ГАС «Правосудие», Почты России, МАКС, Ozon и Т-Банка.
- Добавлена настройка интеграций по запросу и общий preflight без внешних зависимостей.
- Т-Банк поддерживает безопасный базовый режим чтения на macOS и Windows.
- Добавлены проверки конфигурации сервисных навыков и расширена документация первичной настройки.

## 1.0.0 – 2026-07-11

- Первая публичная версия шаблона.
- Первичная настройка владельца и компьютера.
- Базовые навыки задач, проектов и входящих файлов.
- Защита от случайной персонализации публичного репозитория.
